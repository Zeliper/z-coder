## 실행 모드

### 동기 모드 (Sync)
빠른 결과가 필요한 경우 사용합니다. 결과를 즉시 반환합니다.

**특징**:
- 타임아웃: 30초
- 용도: {sync_use_cases}
- 단일 파일/클래스 분석에 적합

**사용 예시**:
```
"PlayerController 클래스 디컴파일해줘"
"이 함수의 심볼 정보 확인해줘"
```

### 비동기 모드 (Async)
시간이 오래 걸리는 작업에 사용합니다. 백그라운드에서 실행됩니다.

**특징**:
- 타임아웃: 없음 (백그라운드 실행)
- 용도: {async_use_cases}
- 대량 처리, 전체 프로젝트 분석에 적합
- 완료 알림: 작업 완료 시 결과 파일 생성

**사용 예시**:
```
"모든 DLL 파일을 디컴파일해줘"
"전체 프로젝트 코드 품질 분석해줘"
```

## 입력
- **필수**: 분석 대상 파일/폴더 경로
- **선택**: 실행 모드 (sync/async, 기본값: sync)
- **선택**: 출력 경로
- **선택**: 추가 옵션

## 작업 흐름

### 동기 모드 흐름
1. 입력 경로 유효성 검사
2. 도구 명령어 실행 (타임아웃 30초)
3. 결과 파싱 및 요약
4. 결과 즉시 반환

### 비동기 모드 흐름
1. 입력 경로 유효성 검사
2. 백그라운드 작업 시작
3. 작업 ID 반환
4. 작업 완료 시 결과 파일 생성
5. 완료 상태 확인 가능

## 오류 처리

### 일반 오류
- **도구 없음**: "도구가 설치되어 있지 않습니다. 설치 후 다시 시도하세요."
- **경로 오류**: "대상 파일/폴더를 찾을 수 없습니다."
- **타임아웃**: "동기 모드 타임아웃. 비동기 모드로 다시 시도하세요."
